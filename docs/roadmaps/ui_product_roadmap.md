# UI / Product Surface Roadmap

This roadmap defines the user-facing product layer for the Land Feasibility Platform.

The UI layer transforms the underlying intelligence system into a usable application for developers, land acquisition teams, and investors.

The UI must allow users to:

- explore parcels
- run feasibility analysis
- visualize subdivision layouts
- evaluate development economics
- compare land opportunities

Master roadmap reference:

bedrock/docs/land_feasibility_roadmap.md

---

# Purpose

Provide a user interface that exposes the capabilities of the feasibility pipeline.

The UI is responsible for:

- parcel discovery
- feasibility analysis initiation
- layout visualization
- financial report presentation
- deal comparison

The UI must make the system usable by non-technical real estate professionals.

---

# Scope

This roadmap includes:

- parcel map exploration
- feasibility analysis interface
- layout visualization
- feasibility reporting
- parcel search and filtering
- deal comparison tools

This roadmap does NOT include:

- parcel ingestion pipelines
- zoning extraction
- layout engine algorithms
- financial modeling logic

Those systems operate beneath the UI layer.

# Core UI Modules

Parcel Explorer

Allows users to search and select parcels.

Feasibility Runner

Allows users to run the pipeline on selected parcels.

Layout Visualization

Displays subdivision layouts generated by the layout engine.

Feasibility Report

Displays the economic evaluation of the development project.

Deal Database

Stores previously analyzed parcels and feasibility results.

---

# Milestone UI-1 — Parcel Map Explorer

Goal

Allow users to browse parcels using a map interface.

Capabilities

- map-based parcel browsing
- parcel search by location
- parcel highlighting
- parcel metadata display

Definition of Done

Users can click a parcel on the map and retrieve parcel information.

---

# Milestone UI-2 — Feasibility Analysis Trigger

Goal

Allow users to initiate the feasibility pipeline.

User action

Select parcel → click "Run Feasibility".

System action

Call:

POST /pipeline/run

Definition of Done

Users can run feasibility analysis for a selected parcel.

---

# Milestone UI-3 — Layout Visualization

Goal

Display subdivision layouts generated by the layout engine.

Visualization elements

- lot boundaries
- road network
- parcel boundaries

Rendering format

GeoJSON overlays on map interface.

Definition of Done

Users can visually inspect the generated subdivision layout.

---

# Milestone UI-4 — Feasibility Report Interface

Goal

Present financial feasibility results to the user.

Report contents

- number of lots
- estimated home price
- construction cost estimate
- development cost estimate
- projected profit
- ROI
- risk score

Definition of Done

Users can view a complete feasibility report for a parcel.

---

# Milestone UI-5 — Deal Comparison Interface

Goal

Allow users to compare multiple parcels.

Comparison metrics

- unit yield
- projected revenue
- projected profit
- ROI
- risk score

Definition of Done

Users can compare feasibility results across multiple parcels.

---

# Milestone UI-6 — Deal Database

Goal

Persist feasibility runs and layouts for later analysis.

Stored objects


DealRecord
parcel_id
layout_id
feasibility_result
timestamp


Capabilities

- retrieve past feasibility runs
- sort opportunities by ROI
- track evaluated parcels

Definition of Done

Users can revisit and compare previously analyzed parcels.

---

# Success Metrics

The UI layer is considered complete when:

- parcels can be explored through a map interface
- feasibility analysis can be triggered from the UI
- layouts are visualized correctly
- financial reports are displayed clearly
- parcels can be compared and ranked
